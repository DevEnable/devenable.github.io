Cross-Platform Mobile Map Application with Xamarin and Azure Mobile Services (.Net) Blog Series

In this blog post I introduce the proof of concept using mobile technologies that I am going to go through in a blog series.

Update: Since I originally started writing this blog series I have bought a Mac-Mini and have expanded the proof of concept to cover an iOS application as well.

Introduction

A while ago I entered a competition being run by a local government organization that were making their data publically available. They wanted ideas on how they could use that data. I saw one piece of data where they listed publically available Internet locations and with the help of some colleagues we submitted an entry that would allow users in the area to find free publically available Internet.

The Proof of Concept

The competition didn't call for any deliverables beyond the idea, and our idea didn't end up being accepted but I'm interested in mobile development and I've been teaching myself Xamarin. I am an individual that learns by doing so I decided to embark on a proof of concept using what I'd learnt about Xamarin and the recently announced the preview version of .Net Mobile Services and I decided to jump right in.

Mobile Service

  • Takes nightly updates from an external remote data source.
  • Takes the addresses in this data and finds the geolocation for the address storing it in a SQL Azure database.
  • Services requests from mobile applications that want data in a certain range around the supplied latitude and longitude.

Mobile Applications (Android, iOS, & WP8)

  • Requests geospatial data from Mobile Services with relevant data around the users current location.
  • Takes that data and displays it on a map with markers and overlay.
  • When the user clicks on a marker it displays additional information.
  • Markers can have special icons categorizing information.

I would have liked to create an iOS application well, however, iOS machine in order to use XCode for I will initially be targeting Android and Windows Phone 8 devices. At some point I will probably just buy a Mac Mini. (I'm aware of the emulator option, I just don't think I want to go there)

In following blog posts I will detail how I created the various components and got the solution working end to end.
